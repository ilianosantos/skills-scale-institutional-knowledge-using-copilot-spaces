## Quick Reference — Role Ownership by Phase

| Role | Initiation | Planning | Execution | Release | Retro |
|---|---|---|---|---|---|
| **Project Manager** | Coordinates kickoff, charter | Owns plan, timeline, risks | Tracks progress, escalates | Release coordination | Facilitates retro, tracks actions |
| **Product Manager** | Defines outcomes, metrics | Owns backlog, roadmap | Clarifies requirements | Stakeholder announcement | Reviews metrics |
| **Developer** | Input on feasibility | Estimation & planning | Implements & reviews code | Deploys | Contributes to retro |
| **UX Designer** | Contributes user insights | Design plan & prototypes | Designs, hands off to Dev | Reviews released UI | Contributes to retro |
| **QA Specialist** | Advises on testability | Defines test strategy | Tests, triages defects | Release readiness sign-off | Contributes to retro |
| **Technical Writer** | — | Documentation plan | Drafts docs | Publishes release notes | Updates process docs |
| **Business Analyst** | Requirements elicitation | Documents requirements | Clarifies requirements | UAT support | Contributes to retro |
| **Stakeholders** | Provide inputs, approvals | Review scope | UAT participation | Approval & announcement | Input on outcomes |
