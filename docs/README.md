# OctoAcme Project Management Docs

This README provides an overview and easy navigation for all project and process documentation in the `docs/` folder.

## Summary of OctoAcme Project Management Processes

OctoAcme organizes work using proven program management and delivery practices. Projects flow through six key phases:

1. **Initiation** — Idea vetting, stakeholder alignment, and scope definition. The project charter is created, success criteria are agreed upon, and initial resources are allocated. See [octoacme-project-initiation.md](./octoacme-project-initiation.md).

2. **Planning** — Backlog creation, risk assessment, milestone setting, and timeline agreement. Dependencies are mapped, and the team aligns on the Definition of Done. See [octoacme-project-planning.md](./octoacme-project-planning.md).

3. **Execution & Tracking** — Agile delivery through iterative sprints, board management, continuous integration, and quality assurance. Progress is tracked against milestones. See [octoacme-execution-and-tracking.md](./octoacme-execution-and-tracking.md).

4. **Risk & Communication** — Active maintenance of the risk register, structured status reporting, and proactive stakeholder communications. Blockers are surfaced and resolved quickly. See [octoacme-risks-and-communication.md](./octoacme-risks-and-communication.md).

5. **Release & Deployment** — Pre-release readiness checks, deployment checklists, rollback protocols, and release announcements. Nothing ships without meeting the agreed release criteria. See [octoacme-release-and-deployment.md](./octoacme-release-and-deployment.md).

6. **Retrospective & Continuous Improvement** — Structured retrospectives to capture learnings, agree on action items, and feed improvements back into future iterations. See [octoacme-retrospective-and-continuous-improvement.md](./octoacme-retrospective-and-continuous-improvement.md).

## How to Use These Docs

- **New to OctoAcme?** Start with the [Project Management Overview](./octoacme-project-management-overview.md) for a one-page summary of our approach, principles, and core roles.
- **Joining an active project?** Check the [Execution & Tracking](./octoacme-execution-and-tracking.md) doc to understand sprint ceremonies, board conventions, and quality expectations.
- **Starting a new project?** Follow the [Project Initiation](./octoacme-project-initiation.md) guide to create a charter, align stakeholders, and define success criteria, then move on to [Project Planning](./octoacme-project-planning.md).
- **Looking for role definitions?** See [Roles & Personas](./octoacme-roles-and-personas.md) for a breakdown of each contributor's responsibilities.
- **Adding docs to Copilot Spaces?** Place process-specific documents into `.copilot/` so that Copilot Spaces can use them as context for your team's workspace.
